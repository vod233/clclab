GitHub Path · 学习路线

从 第一次 commit，
到自动部署。

一条循序渐进的 GitHub 学习路线，七个阶段，每个阶段只做一件明确的事。 配合下面项目实验区的小项目一起练，会更快。

01
Git 基础

add / commit / push / pull，理解工作区、暂存区与版本库。

02
分支管理

用 feature branch 隔离每次改动，掌握 merge 与 rebase 的取舍。

03
Pull Request

写好 description，配置 review 规则，把 PR 变成团队的协作语言。

04
Issue 管理

用 issue 记录需求与 bug，labels / milestones 让进度可追踪。

05
README 写作

项目门面：标题、截图、运行步骤、技术栈，让别人 60 秒内看懂。

06
GitHub Actions

用 workflow 文件跑测试、构建、格式化，让 CI 接管机械劳动。

07
自动部署

push 即部署：GitHub Pages / Cloudflare Pages / VPS，配置一次，长期生效。
